Isolated scopes in Angular Directives

Isolated scopes are heavily used in angular directives to make directives resuable. By default directives access scope of their parent. Changing value in directives changes value in parent scope. We can define scopes in directives in three ways:-

  • Set scope property in directives to false(default value of directives) which allows to use scope of their parent.
  • Set scope property in directives to true which allows to create their own scope by using prototypical inheritance.
  • Set scope property to an object with '@'(attribute binding), '='(two way binding) and '&'(expression binding).

We disscuss bellow about last one i.e binding properties of directives.

  • @: Attribute Binding

    Attribute binding allow directives to simply pass value from parent scope to directives scope. This is one-way binding. Changes value in directives will not change value in parent.
